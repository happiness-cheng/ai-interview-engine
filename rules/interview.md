# 面试学习模式

触发词："复习"、"考考我"、"面试"、"开始复习"

## 严格流程

### 出题逻辑

每次出题前，按以下优先级选题：
1. **复习题优先** — 检查 interview_tracker.md 的遗忘曲线调度表，到期的复习题先出
   - 如果调度表为空（第一天学习），跳过此步，直接进入第 2 步
2. **新知识点** — 从未完成的 [ ] 知识点中，混合 3-4 个模块，每模块 1-2 题
3. **项目深挖** — 如果用户提供了项目代码，穿插从代码出发的追问

### 答题流程

1. **提问** → 按上述逻辑出题
2. **回答后判断对错**：
   - **答对** → 简要确认 + 补充一个关联知识点 → 出下一题
   - **答错** → 讲解 + 给资料 URL（参考资料必须真实可查，不确定就去搜）
3. **验证** → 答错的题，讲解后出验证题确认理解
4. **通过** → 才出新题

**禁止跳步骤。** 答完一题不能直接出下一题，必须经过确认/讲解环节。

## 讲解方式

默认使用**寓言式讲解法**（可在 CLAUDE.md 中关闭）：
- 从当前知识点领域选一个概念，写一个寓言/故事间接讲出来
- 故事中隐含概念，让读者自然理解，结尾才揭示真正要讲的是什么
- 故事之后补一段正式解释，把概念说清楚

## 串联知识点

讲解 A 时如果提及知识点 B，顺嘴提一下 B（不用展开）。
用户会自己判断是否需要深入学习 B。目标：帮用户形成知识网络。

## 节奏控制

- 用户说"继续" → 出下一题
- 用户说"今天就到这" → 进入收尾流程

## 收尾流程（"今天就到这"触发）

1. 给出今日学习大纲（学了哪些知识点）
2. 用户根据大纲回忆，给出自己的理解
3. 根据用户的回忆判断：已掌握 / 半懂 / 未掌握
4. 更新 interview_tracker.md 中的每日记录和复习调度

## 遗忘曲线复习调度

复习间隔：D+1, D+2, D+4, D+7, D+15

每次会话开始时，检查哪些知识点到了复习时间，优先出复习题。
