# Tracker 规范

## 文件命名

```
knowledge/trackers/{主题}.md
```

每个主题一个独立文件。

---

## Tracker 模板

完整模板见 [templates/tracker.md](../templates/tracker.md)。

核心结构：

```markdown
# {主题} 学习追踪系统

## 学习目标

（记录目标岗位、考试、学习范围或用户目标）

## 模块定义

| 编号 | 模块 | 简写 | 优先级 |
|------|------|------|--------|

## 知识点清单

标记说明：
- `[ ]` 未学习
- `[~]` 学习中
- `[x]` 已理解
- `[✓]` 已巩固
- `[★]` 已掌握

## 复习调度表

| 知识点 | 状态 | 复习阶段 | 上次评分 | 上次学习 | 下次复习 | 薄弱点 | 待验证 |
|--------|------|----------|----------|----------|----------|--------|--------|

## 知识网络与后续支线

（每个重要知识点记录已建立连接和待深化方向）

## 每日记录
```

---

## 更新时机

每道关键题后立即更新，不等待用户说"今天就到这"。

关键题 = 产生正式评分、改变状态、暴露薄弱点、完成待验证、打开/完成重要支线、完成实践证据。

---

## 每日记录格式

```markdown
### 2026-06-20

- 一致性哈希：
  - 首次讲解：完成
  - 自我解释：通过
  - 取模哈希对比：Good
  - 删除节点迁移比例：Hard
  - 下次重点：删除节点场景
```

不需要把每一次完整对话全部复制进 tracker。

---

## 知识网络记录格式

```markdown
## 知识网络与后续支线

### std::variant

已建立连接：
- std::variant → union：来源与类型安全
- std::variant → Lambda：visit 的调用方式

待深化：
- std::variant → Visitor 模式
- std::variant → optional / any / expected
- std::variant → 虚函数多态
```

---

## 即时理解确认记录

微型教学后立即答对，只算即时理解确认：

```markdown
- 不完整类型：
  - 本次状态：首次教学
  - 即时理解：通过
  - 正式评分：待延迟验证
  - 与主线关系：解释 Pimpl 析构函数为什么放在 cpp
```

---

## 维护规则

防止 tracker 无限膨胀：

- 相同知识连接合并，不重复记录
- 每个锚点只保留最重要的 3～8 条连接
- 已正式成为知识点的支线，移入知识点清单，不在"待深化"重复保留
- 已完成的旧待验证任务定期归档
- 每日记录保留结论，不复制完整问答
