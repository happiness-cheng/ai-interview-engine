# 知识网络规则

## 核心理念

不是把知识点一个个孤立背下来，而是以当前知识为中心，沿着相关关系向外发散，逐步形成知识网络。

---

## 锚点与发散

每次学习先确定一个**主知识点**（锚点），然后沿高价值关系发散：

| 关系类型 | 说明 | 示例 |
|----------|------|------|
| 因果关系 | 为什么会出现这个知识 | 编译依赖、头文件污染 |
| 前置关系 | 理解它还需要知道什么 | 前向声明、不完整类型 |
| 实现关系 | 它底层依赖哪些机制 | unique_ptr、RAII |
| 对比关系 | 它和相似方案有什么区别 | 抽象类、接口类 |
| 风险关系 | 使用它会带来什么问题 | 堆分配成本、指针间接访问 |
| 应用关系 | 项目中在哪里使用 | 库开发、SDK |
| 面试关系 | 面试官通常怎样继续追问 | 拷贝语义、移动语义 |

---

每打开一条支线，必须明确说明：

> 这个知识为什么和当前主线有关。

例如：

> 这里需要补一下 `unique_ptr`，因为 Pimpl 通常要让当前对象独占自己的实现对象，并在析构时自动释放。

---

## 发散时先判断知识状态

### 已经学过

可以直接用面试问题提取，并正常评分。

### 学过但不稳定

先做一个简短回忆或补充，再继续连接主线。

### 完全没学过

先切换成一个短小的"微型教学"：

```
直观解释
→ 正式概念
→ 它和主线的关系
→ 一个简单确认问题
```

第一次不知道不能判 Again。

---

## 核心通过后必须主动扫描知识网络

核心理解通过后，不能立即宣布"已掌握"，必须主动进行一次知识网络扫描。

### 扫描流程

```
核心知识通过
↓
生成 3～6 个最相关连接
↓
说明它们为什么有关
↓
区分已学 / 薄弱 / 未学
↓
选择 1～2 条高价值支线展开
↓
支线完成后回到主线整合
↓
其余支线登记为后续学习方向
```

### 适用条件

- 核心概念、高频面试知识、高关联知识：必须扫描
- 简单事实、低关联知识、纯记忆内容：可只建 1～2 个必要连接，或跳过支线展开

---

## 支线分类处理

### 第一类：必须现在补的前置知识

没有它，就无法继续理解当前主线。

处理：立即用微型教学补上，讲完马上回到主线。

### 第二类：有帮助，但不是立刻必需的关联知识

能帮助理解，但继续深入可能偏离主线。

处理：讲清它和当前知识的连接，详细内容留给它自己的知识点。

### 第三类：高级延伸知识

可能被高级面试追问，但第一次接触时不一定必须全部深入。

处理：登记为后续深化，等主线稳固后再展开。

---

## 支线何时升级为独立知识点

支线具备以下条件时，加入知识点清单并建立独立调度行：

- 有独立定义和独立面试价值
- 需要单独跨日复习

仅用于帮助理解主线的局部细节，只记录在知识连接或薄弱点中，不单独调度。

---

## 支线学完后必须回到主线整合

补完支线知识后，必须明确收回来：

> 现在回到主线：正因为 `unique_ptr` 表达独占所有权并自动释放资源，所以它很适合作为 `pImpl_` 的类型。

---

## 停止条件

不是由问题数量决定，而是看：

- 用户是否仍然跟得上
- 关联知识是否直接有价值
- 是否开始明显过载
- 是否已经偏离当前主线太远
- 本轮是否出现太多首次接触的新概念

如果连续出现多个"不知道"，说明已经进入新的知识区域，应暂停考试模式，切换为教学模式。

## 知识网络示例

### Pimpl 模式

```
Pimpl
├─ 为什么需要它
│  ├─ 编译依赖
│  ├─ 头文件污染
│  └─ ABI 稳定
├─ 怎么实现
│  ├─ 前向声明
│  ├─ 不完整类型
│  └─ unique_ptr
├─ 资源管理
│  ├─ RAII
│  ├─ 析构函数
│  └─ double free
├─ 对象语义
│  ├─ 拷贝构造
│  ├─ 深拷贝
│  ├─ 移动构造
│  └─ Rule of Five
└─ 实际取舍
   ├─ 堆分配成本
   ├─ 指针间接访问
   ├─ 内联受限
   └─ 适合库和 SDK
```

### std::variant

```
std::variant
├─ 来源与底层概念
│  ├─ union
│  ├─ 活动成员
│  └─ 类型标签/index
├─ 访问方式
│  ├─ std::get
│  ├─ std::get_if
│  ├─ std::holds_alternative
│  └─ std::visit
├─ visit 机制
│  ├─ Lambda
│  ├─ 泛型 Lambda
│  ├─ 函数重载
│  └─ 编译期穷尽检查
├─ 相似类型
│  ├─ std::optional
│  ├─ std::any
│  └─ std::expected
└─ 实际应用
   ├─ JSON 值
   ├─ 消息与事件系统
   └─ 状态机
```
