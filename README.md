## 你一定经历过这些

用 ChatGPT / Claude 准备面试，翻车场景千篇一律：

| 场景 | 你期望的 | 实际发生的 |
|------|---------|-----------|
| 答错了一道题 | 指出错误 + 讲清楚 | "Great answer!" 然后下一题 |
| 第二天继续学 | 记得昨天学了什么 | 从零开始，你得复述一遍 |
| 学了一周 | 知道该复习什么了 | 没有计划，随机抽题 |
| 看完讲解 | 面试时能说出来 | 看懂了 ≠ 会说了 |
| 学了很多知识点 | 知道它们之间的关系 | 孤岛式记忆，串不起来 |

**根本原因：ChatGPT 是一个贴心的陪聊，不是一个严格的老师。**

它不会纠正你，不会追踪你的进度，不会在你遗忘的临界点安排复习，更不会在你答错后用变式题验证你是否真的懂了。

## 本项目的解法：用规则文件强制 AI 当严师

Claude Code 有一个独特能力：**规则文件（CLAUDE.md / SKILL.md）+ 持久化文件系统**。

通过一套精心设计的规则，我们把 Claude Code 从"贴心陪聊"改造成"严师"：

### 核心卖点

**1. 分层释放教学 — 不是一股脑倒给你**

```
第一轮：直观模型 → 正式定义 → 一个核心机制 → 一个确认问题
第二轮：为什么有效 → 最小有效示例 → 一个追问
第三轮：优缺点/对比 → 面试追问 → 知识网络支线
```

不是一口气把定义、机制、优缺点、代码全倒给你。每一层确认你理解了，才进入下一层。就像一个好老师，看到你点头了才继续讲。

**2. 知识网络 — 学一个点，连一张网**

学 `Pimpl 模式` 时，自动扫描关联知识：

```
Pimpl
├─ 为什么需要它 → 编译依赖、头文件污染、ABI 稳定
├─ 怎么实现 → 前向声明、不完整类型、unique_ptr
├─ 资源管理 → RAII、析构函数、double free
├─ 对象语义 → 拷贝构造、深拷贝、移动构造、Rule of Five
└─ 实际取舍 → 堆分配成本、指针间接访问、适合库和 SDK
```

不是孤立地背知识点，而是以当前知识为中心，沿因果、前置、实现、对比、应用、面试六种关系向外发散。面试官追问任何方向，你都有准备。

**3. 四级评分 + 区分错误类型 — 不是简单的对/错**

| 评分 | 含义 | 后续动作 |
|------|------|---------|
| **Again** | 核心错误或完全忘记 | 重新讲解 → 变式验证 → 跨日验证 |
| **Hard** | 方向对但关键遗漏 | 区分内容型（后退一级）vs 表达型（不后退，补术语） |
| **Good** | 核心正确，逻辑完整 | 正常前进 |
| **Easy** | 稳定、准确、能迁移 | 可跳级 |

关键区别：**表达型 Hard 不后退**。你理解正确但术语不标准，不会被当成没学会。很多面试准备工具只看"对不对"，不看"为什么错"。

**4. 未学内容不评分 — 不冤枉你**

首次接触的知识点，你回答"不知道"，不会被判 Again。系统会切换为微型教学，讲完再确认。

这解决了一个常见问题：面试准备工具把"没学过"和"学过但忘了"混为一谈，导致复习计划失真。

**5. 延迟变式验证 — 不是原题重问**

答错后不会立即重问原题。系统会：
- 穿插其他内容
- 用变式题验证（换角度、换场景）
- 次日跨日再次验证

这比"答错了 → 马上再问一遍"有效得多。你不是记住了答案，而是真正理解了概念。

**6. 遗忘曲线复习 — 在你快要忘记时复习**

```
1天 → 3天 → 7天 → 14天 → 30天 → 60天
```

复习角度随时间变化：
- 1天：检查核心定义和机制
- 3天：检查原因和对比
- 7天：检查边界和变式
- 14天+：检查面试追问、项目应用、综合设计

不是每次复习都问"什么是 X？"，而是随着掌握程度加深，考察角度越来越实战。

**7. 主线/支线分离评分 — 不连坐**

你已经理解了 `std::variant`，但不知道 `Lambda`。系统不会因为 Lambda 不会就把 variant 也判成 Again。主线和支线分别评分、分别调度。

**8. 跨会话持久化 — 每次继续，不是从零开始**

学习进度保存在文件中。下次打开 Claude Code，系统读取 tracker，自动检查哪些知识点该复习，哪些待验证还没完成。

## 快速开始

### 1. Fork 仓库

```bash
gh repo fork happiness-cheng/ai-interview-engine --clone
```

或者直接下载 ZIP 解压到任意目录。

### 2. 用 Claude Code 打开

```bash
cd ai-interview-engine
claude
```

### 3. 开始第一次对话

输入：

> 我要准备 C++ 后端开发面试，目标是大厂实习。开始复习。

Claude 会读取规则文件和题库，开始第一轮教学。

### 4. 自定义你的内容

- **换技术栈**：复制 `knowledge/TEMPLATE.md` → `interview_tracker.md`，填入你的知识点
- **加项目**：在 `interview_tracker.md` 的 M2 模块填入你的项目关键设计决策
- **调规则**：编辑 `references/` 下的文件修改教学方式、评分标准等

```
ai-interview-engine/
├── CLAUDE.md                         # 入口：全局规则 + 使用说明

├── README.md                         # 你正在看的这个文件

├── references/                       # 详细规则（按需加载）
│   ├── teaching.md                   # 教学规则：分层释放、直观模型、面试追问链

│   ├── tracker.md                    # Tracker 规范：文件格式、更新时机、维护规则
```

## 使用场景

- **秋招/春招技术面试** — 系统化覆盖高频考点，按遗忘曲线巩固
- **转行/跳槽面试** — 针对新领域建立知识网络，不只是背八股
- **期末考试复习** — 换知识点库即可，引擎逻辑通用
- **项目复盘** — 从代码出发深挖设计决策，准备面试中的项目追问

## 工作原理

### 教学流程

```
新知识：分层释放（直观→定义→机制→确认→示例→追问→网络）
旧知识：先测后讲（无提示提问→评分→针对薄弱点反馈）
答错：  延迟变式验证（不原题重问）→ 跨日再次验证
```

### 评分与复习

```
评分：Again / Hard(内容型|表达型) / Good / Easy
复习：1天 → 3天 → 7天 → 14天 → 30天 → 60天
角度：核心定义 → 原因对比 → 边界变式 → 面试实战
```

### 知识网络

```
锚点（当前知识）→ 六种关系发散 → 区分已学/薄弱/未学 → 选择高价值支线展开
```
