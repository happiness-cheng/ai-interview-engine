## 你一定经历过这些

用 ChatGPT / Claude 准备面试：

- 答错了，它说 "Great answer!" 然后下一题
- 第二天打开新对话，它不记得你昨天学了什么
- 学了一堆知识点，不知道什么时候该复习
- 看完讲解觉得懂了，面试时却说不出来

**根本原因：ChatGPT 是陪聊，不是老师。** 它不纠错、不记进度、不安排复习、不验证你是否真懂了。

## 一个典型的学习闭环

下面是按设计规则编写的典型交互示例：

### 普通 AI 的做法

```
你：什么是 Pimpl？
AI：Pimpl（Pointer to Implementation）是一种 C++ 设计模式，
    通过将实现细节隐藏在指针后面来减少编译依赖...
    [一大段定义 + 代码 + 优缺点 + 使用场景，一次全倒给你]
```

你点头说"懂了"。三天后面试官问"为什么 Pimpl 要用 unique_ptr"，你说不出来。

### AI Interview Engine 的做法

```
你：什么是 Pimpl？

AI：你点外卖的时候，只看菜单和价格，不需要知道厨房里有几个人、
    用什么锅。菜单不变，厨房随便换。

    Pimpl 就是这个思路：你只暴露"接口"，把"实现"藏起来。

    正式定义：Pimpl 将类的实现细节放到一个独立的实现类中，
    通过指针间接访问，从而隔离编译依赖。

    一个核心机制：头文件只看到一个指针，看不到实现类的成员，
    所以改实现不会触发重新编译。

    确认一下：你知道为什么"看不到成员"就能减少重编译吗？
```

你回答后，它根据你的理解程度决定下一步。三天后继续时：

```
AI：上次你理解了 Pimpl 的基本目的，但拷贝语义还不稳定。
    今天先验证这一点：如果 Pimpl 类需要值语义，拷贝构造函数
    该怎么写？为什么不能用默认的？
```

**这就是区别：它记得你哪里弱，并且针对弱项复习。**

## 三个核心能力

### 1. 真正教会你

不是一股脑倒给你，而是分层释放：

```
第 1 轮：建立主干
  直观模型 → 正式定义 → 一个核心机制 → 确认问题
第 2 轮：理解与应用
  为什么有效 → 最小有效示例 → 一个追问
第 3 轮：面试深化
  优缺点/对比 → 面试追问 → 知识网络支线
```

关键设计：
- **首次接触不会的，不判错** — 区分"没学过"和"学过但忘了"
- **类比负责理解，正式模型负责纠错** — 你说"菜单不变厨房随便换"，它先确认理解正确，再补充标准术语
- **一次只问一个问题** — 不会一次倾倒全部内容

### 2. 真正判断你会不会

不是简单的"对/错"，而是四级评分：

| 评分 | 含义 | 后续动作 |
|------|------|---------|
| **Again** | 核心错误或完全忘记 | 换方式重新讲解 → 变式验证 → 次日再验 |
| **Hard** | 方向对但关键遗漏 | 区分内容型（后退一级）vs 表达型（补术语，不后退） |
| **Good** | 核心正确，逻辑完整 | 正常前进 |
| **Easy** | 稳定、准确、能迁移 | 可跳级 |

关键设计：
- **答错不原题重问** — 穿插其他内容后用变式题验证
- **主线/支线分别评分** — 你理解 variant 但不会 Lambda，不会连坐 variant
- **复习角度随阶段变化** — 1 天查定义，3 天查原因，7 天查边界，14 天+ 查实战

### 3. 真正记得你学到哪里

学习进度保存在文件中，跨会话持续：

```
第 1 天：Pimpl 核心机制 → Good
第 3 天：再次启动，自动检查到期复习 → 拷贝语义 → Hard
第 7 天：再次启动 → "上次拷贝语义不稳定，今天先验证这一点"
```

关键设计：
- **知识网络** — 学 Pimpl 时自动连接 unique_ptr、RAII、Rule of Five
- **间隔复习** — 1天 → 3天 → 7天 → 14天 → 30天 → 60天
- **Tracker 持久化** — 每道题评分后立即写入，不攒到收尾

## 快速开始

### 方式一：Fork 整个仓库（推荐）

```bash
gh repo fork happiness-cheng/ai-interview-engine --clone
cd ai-interview-engine
claude
```

然后输入：

> 我要准备 C++ 后端开发面试，目标是大厂实习。开始复习。

### 方式二：安装为独立 Skill

把 `.claude/skills/interview/` 复制到你的项目中：

```bash
cp -r .claude/skills/interview /your/project/.claude/skills/
```

然后在你的项目中启动 Claude Code，输入：

```
/interview C++ 后端，大厂实习
```

> **区别**：方式一是完整学习工作区，自带 C++ 示例知识地图、用户画像和完整工作目录；首次开始学习时自动创建个人 Tracker。方式二是独立 Skill 包，自带完整规则和示例；首次运行时会根据目标自动创建 Tracker，但不会预置你的知识范围。

## 自定义

### 换技术栈

1. 复制 `.claude/skills/interview/templates/tracker.md` 为 `knowledge/trackers/{你的主题}.md`
2. 填入你的模块和知识点
3. 告诉 Claude "我要准备 {你的主题} 面试"

示例：`knowledge/trackers/Java微服务.md`、`knowledge/trackers/前端React.md`

或者直接告诉 Claude 你的目标岗位，让它帮你生成初始 Tracker。

### 加项目深挖

在 Tracker 的 M2 模块填入你的项目关键设计决策。面试官会从简历挑细节追问，这里准备的是"为什么这样设计"而不是"用了什么技术"。

### 调规则

编辑 `.claude/skills/interview/references/` 下的文件修改教学方式、评分标准、知识网络扫描范围等。

## 方案差异

| 方案 | 擅长什么 | 主要局限 |
|------|---------|---------|
| 普通 AI 对话 | 即时讲解、自由问答、灵活探索 | 无进度追踪、无复习调度、不纠错、答错了可能夸你对 |
| Anki | 间隔重复、记忆卡片、成熟调度算法 | 不擅长动态讲解和连续追问，需要手动制卡 |
| NotebookLM | 基于文档的问答、多文档关联 | 无复习闭环、无评分体系、不追踪个人进度 |
| **AI Interview Engine** | 教学、追问、评分、知识网络、间隔复习统一调度 | 依赖 Claude Code，需要维护知识库，不适合纯碎片时间 |

## 使用场景

- **秋招/春招技术面试** — 系统化覆盖高频考点，按间隔复习巩固
- **转行/跳槽面试** — 针对新领域建立知识网络，不只是背八股
- **期末考试复习** — 换知识点库即可，引擎逻辑通用
- **项目复盘** — 从代码出发深挖设计决策，准备面试中的项目追问

## 已知限制

- **依赖 Claude Code** — 需要 Claude Code CLI 或桌面版，不支持纯网页版 ChatGPT
- **存在使用成本** — 需要 Claude 订阅、Anthropic Console API 额度或受支持的第三方服务，具体计费方式取决于登录和接入方式
- **Tracker 为纯文本** — 没有图形化界面，进度查看需要打开文件
- **单用户设计** — 不支持多人协作学习
- **评分依赖模型判断** — 评分一致性取决于 Claude 的理解，可能有波动
- **无自动测试** — 教学流程的正确性依赖规则文件的设计，目前没有自动化回归测试

## 设计原则

1. **分层释放** — 一次只给一层，确认后再给下一层。防止信息过载。
2. **区分不知道和忘了** — 首次接触不会 ≠ 学过但忘了。这两种情况的处理方式完全不同。
3. **区分理解和表达** — 你理解正确但术语不标准，不会被判为没学会。
4. **不原题重问** — 答错后用变式验证，不是让你背答案。
5. **主线不连坐** — 支线答错不影响主线已取得的成绩。
6. **间隔复习角度渐变** — 随着掌握程度加深，考察角度从定义→原因→边界→实战递进。
