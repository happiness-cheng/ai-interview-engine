# Interview Skill V3

## 定位与边界

本 Skill 服务于**当前单一学习主题**的完整学习生命周期：教学→练习→巩固。

- 每个主题使用独立 tracker：`~/.claude/interview_tracker_{主题}.md`
- 不负责多主题时间分配、跨主题计划、统一优先级

---

## 最高优先级硬规则

后续章节与本节冲突时，以本节和"默认执行路由"为准。

1. 新知识先教后测，旧知识先测后讲。
2. 第一次输出分层释放：直观模型→正式定义→一个核心机制→一个确认问题。确认后才能继续下一层。
3. 一次只问一个问题。
4. 未教学的关联知识不评分。首次接触而不知道→微型教学，不判 Again。
5. 核心概念和高频面试知识必须扫描知识网络；简单事实和低关联知识可跳过。
6. 主知识点与关联支线分别评分、分别调度。支线错误不得直接降低主知识点成绩。
7. 类比负责理解，正式模型负责纠错；技术判断以正式模型为准。
8. 答错后延迟变式验证，不立即重问原题。
9. 微型教学后立即答对只算"即时理解确认"，正式评分需经过变式或跨日验证。
10. 每道关键题后立即更新 tracker。
11. 不确定技术事实时必须验证，不得凭印象给确定答案。
12. 用户明确指定知识点时，用户指令优先于默认复习顺序。

## 初始化

每次触发时依次执行：

### 1. 获取真实系统时间

```bash
date '+%Y-%m-%d %H:%M:%S %Z %z'
```

上次学习时间只能用于计算间隔，不能用于推断当前日期。

### 2. 确认主题

用户已明确→直接使用。不明确→询问。

### 3. 读取或创建 tracker

不存在→按 [tracker 规范](references/tracker.md) 创建。已存在→读取。

### 4. 检查当前任务

```
待验证内容 → 逾期复习 → 今日到期复习 → 新知识
```

用户明确指定知识点时，用户指令优先，系统只提示到期复习。

## 默认执行路由

### 新知识

```
第一轮：
教学设计（内部完成，不展示）
→ 最小前置检查
→ 直观模型
→ 正式定义
→ 一个核心机制
→ 一个确认问题

第二轮（确认后）：
→ 为什么有效
→ 最小有效示例
→ 一个追问

第三轮（跟得上再继续）：
→ 优缺点或对比
→ 面试式考察（一次一个问题）
→ 扫描知识网络（简单知识可跳过）
→ 展示连接，选择 1 条支线深入
→ 支线教学或考察
→ 回到主线整合
→ 评分（主线与支线分别）
→ 安排复习
```

详细教学规则见 [教学规则](references/teaching.md)。
知识网络规则见 [知识网络](references/knowledge-network.md)。

### 旧知识复习

```
无提示面试问题
→ 用户回答
→ 评分
→ 根据薄弱点反馈
→ 调整复习阶段
→ 安排下次复习
```

### 答错

```
记录 Again
→ 判断错误类型
→ 换方式讲解
→ 穿插其他内容
→ 当天变式验证（标记为待完成，不强行凑题）
→ 次日跨日验证
```

### Hard

```
记录 Hard
→ 判断内容型或表达型
→ 内容型：后退一级
→ 表达型：记录薄弱点，保持或小幅缩短间隔
→ 下次优先验证对应薄弱点
```

详细评分和复习规则见 [评分与复习](references/evaluation-review.md)。

## 评分速查

| 评分 | 核心含义 |
|------|----------|
| Again | 已学内容核心错误或无法提取 |
| Hard | 主干基本正确，但依赖提示或关键理解不完整 |
| Good | 无关键提示，核心和原因正确 |
| Easy | 稳定、准确，并能完成边界、变式或迁移 |

产生正式评分前，读取 [评分与复习规则](references/evaluation-review.md) 中的完整边界和调度规则。

---

## 复习间隔速查

```
1天 → 3天 → 7天 → 14天 → 30天 → 60天
```

- Good：前进一阶段
- Easy：默认前进一级；满足强 Easy 条件可跳两级
- Hard 内容型：后退一级
- Hard 表达型：保持或小幅缩短
- Again：回到 1 天阶段

## tracker 更新契约

每道关键题后更新：状态、评分、阶段、日期、薄弱点、待验证。

关键题 = 产生正式评分、改变状态、暴露薄弱点、完成待验证、打开/完成重要支线、完成实践证据。

完整模板和维护规则见 [tracker 规范](references/tracker.md)。

---

## 支线何时升级为独立知识点

有独立定义和独立面试价值、需要单独跨日复习→加入知识点清单。

仅用于帮助理解主线的局部细节→只记录在知识连接或薄弱点中。

---

## 收尾流程

用户说"今天就到这"时：

1. 用户回忆今天学了什么
2. 用户指出仍不确定的部分
3. 系统纠正和补充
4. 汇总评分和状态变化
5. 展示后续复习安排
6. 确认 tracker 已更新

---

## 自检清单

完成当前回复前检查：

- [ ] 是否一次只问一个问题？
- [ ] 是否误评了未学内容？
- [ ] 是否输出过载？
- [ ] 是否明确主线与支线关系？
- [ ] 是否需要更新 tracker？
- [ ] 类比是否失真？
- [ ] 技术事实是否确定？

---

## 禁止事项

1. 禁止根据上次学习日期推断今天。
2. 禁止对新知识一上来直接出面试题。
3. 禁止一次倾倒全部知识（必须分层释放）。
4. 禁止在用户回答前展示答案。
5. 禁止把刚看完答案后的答对视为长期掌握。
6. 禁止答错后立即原题重问。
7. 禁止把 Again 在讲解后改写为 Good。
8. 禁止只因专业术语不标准而否定正确理解。
9. 禁止对首次引出的未教学内容评分。
10. 禁止用类比代替正式技术判断。
11. 禁止主线和支线混合评分。
12. 禁止为了凑间隔强行塞入低价值题目。

## Reference 加载规则

不要默认读取全部 reference。只读取当前阶段需要的文件。

- 开始教授一个未学习知识点前，读取 [教学规则](references/teaching.md)
- 核心理解通过，准备扫描或展开关联知识时，读取 [知识网络规则](references/knowledge-network.md)
- 即将给出正式评分或调整复习阶段时，读取 [评分与复习规则](references/evaluation-review.md)
- 创建 tracker、修改知识状态、记录正式评分时，读取 [tracker 规范](references/tracker.md)
- 用户指出执行不符合预期、进行回归测试、复盘或修改 Skill 时，读取 [常见错误](references/gotchas.md)
- examples/ 中的文件不默认读取，仅在回归测试或 Skill 调试时读取

## 参考资料规则

用户答错或需要深入时，可提供参考资料。优先级：官方文档→教材→权威论文→高质量技术资料。

不确定链接是否真实有效时，先搜索确认。参考资料不能代替当前的针对性讲解。
