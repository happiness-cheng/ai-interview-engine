## 背景

用户是 C++ 后端开发，正在准备秋招面试。之前学过 union，但没学过 variant。

---

## 第一轮输出

**直观模型**：

你有一个盒子，可以放不同类型的东西：笔、苹果、书。但同一时间只能放一样东西。

传统 union 的问题：不告诉你里面现在放的是什么。你伸手进盒子，以为摸到的是苹果，其实里面是笔。

**正式定义**：

`std::variant` 是 C++17 的类型安全 union。它记住里面现在是什么类型。

**一个核心机制**：

variant 内部存了一个索引值（0、1、2），表示"现在存的是第几个类型"。

**一个确认问题**：

> `std::variant` 比传统 `union` 多记了什么信息？

---

## 用户回答后

如果用户答对（"存储数据的类型"）→ 评分 Good，继续第二轮。

如果用户答错 → 讲解，不判 Again（因为是新知识）。

## 第二轮输出

**为什么有效**：

编译器看这个索引，就知道该按哪个类型来读。如果你乱读，会报错或抛异常。

**最小有效示例**：

```cpp
std::variant<int, float, std::string> v;
v = 42;
int val = std::get<int>(v);  // 正确
// std::get<float>(v);  // 抛异常！因为里面不是 float
```

**一个追问**：

> `std::visit` 是怎么知道该调用哪个函数的？
